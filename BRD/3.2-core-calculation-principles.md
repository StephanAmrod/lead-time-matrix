# 3.2. Core Calculation Principles

The system must enforce the following principles:

1. **Lead times must always be calculated at jobcard level first**

2. **All calculations must follow a Base + Additive Components model**

3. **Overrides and adjustments must be applied before any add-ons**

4. **Add-ons must be:**
   - Independent
   - Additive
   - Non-overlapping

5. **Internal and Client lead times must:**
   - Share the same base calculation
   - Differ only where explicitly defined

6. **Missing values must default to TBA**

7. **All calculations must be deterministic and repeatable**

8. **The design must support adding new components without changing the calculation structure**
