# 2.2. Branding Departments Manager

Production department lead times can vary, so a list of branding departments is required to view print codes by department, as well as to maintain invoice and setup codes.

## 1. Search and Filter

| Description | Detailed Requirements |
|---|---|
| **By Name** | Users must be able to search for a branding department by name. This search must support partial matches and return all relevant results. |
| **By Print Code** | Users must be able to filter by print code.<br/>The search must support partial matching.<br/>The system must return all matching results. |

---

## 2. View Branding Departments

Branding departments must be displayed in a hierarchical structure.

| Description | Detailed Requirements |
|---|---|
| **Level 1: Branding Departments** | • The system must display a list of all branding departments.<br/>• For each branding department, the following must be displayed:<br/>◦ Department name<br/>◦ Additional Charges<br/>• Each branding department must be expandable to display print codes linked to that branding department. |
| **Level 2: Print Codes** | • Each branding department must contain print codes. For each group, the following must be displayed:<br/>• Name<br/>• Description<br/>Each branding group must be expandable to display invoice and setup codes. |
| **Level 3: Invoice Codes and Setup Codes** | Each branding group must contain invoice codes and setup codes. Each code must include:<br/>• Code<br/>• Description<br/>• Colours<br/>• Quantity |

---

## 3. Manage Branding Departments

### Additional Charges Codes

| Description | Detailed Requirements |
|---|---|
| **Add** | • Users must be able to define additional charges codes per branding department.<br/><br/>**Validation rules:**<br/>• Multiple additional charges may exist per branding department.<br/>• The code must be unique.<br/>• Duplicate codes must not be allowed. If a duplicate code is entered, the system must prevent the action and display an appropriate validation message. |
| **Delete** | Users must be able to delete a code. |
| **Edit** | • Users must be able to update an existing code.<br/><br/>**Validation rules:**<br/>The updated code must not duplicate an existing code. If a duplicate code is entered, the system must prevent the update and display an appropriate validation message. |
