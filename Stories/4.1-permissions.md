# Story: Lead Time Matrix - I want role-based permissions to control access to module features

**BRD References:** 4.1, 4.2, 4.3, 4.4, 4.5
**Dev Hours:** 16

---

## Overview

Access to the Lead Time Matrix module must be role-based and controlled through permissions. This story implements all permissions required to secure the module's features across Special Dates Manager, Branding Departments Manager, Production Lead Time Matrix, and Warehouse Lead Time Matrix.

> **Important:** All deletion permissions must be selectively allocated to specific users.

---

## Implementation Pattern

Follow the existing Moyo authorization pattern as implemented in other modules (e.g., StockChecking):

1. **Claims Class** - Define all claims as const strings in `Moyo.Common.Domain.Claims.LeadTimeMatrix.LeadTimeMatrixClaims`
2. **Authorization Registration** - Register endpoint handlers with claims in `LeadTimeMatrix.Api.Authorization.LeadTimeMatrixAuthorizationRegistration`

## Authorization Registration

Create the authorization registration class at `LeadTimeMatrix.Api.Authorization.LeadTimeMatrixAuthorizationRegistration`:

```csharp
using Moyo.Common.Application.Authorization;
using static Moyo.Common.Domain.Claims.LeadTimeMatrix.LeadTimeMatrixClaims;

namespace LeadTimeMatrix.Api.Authorization;

/// <summary>
/// Partial class for registering Lead Time Matrix authorization rules.
/// </summary>
public static partial class LeadTimeMatrixAuthorizationRegistration
{
    /// <summary>
    /// Registers all Special Dates authorization rules.
    /// </summary>
    private static void RegisterSpecialDatesEndpoints(EndpointAuthorizationPolicyBuilder builder)
    {
        // GET /api/LeadTimeMatrix/SpecialDates
        builder.RegisterHandler<GetSpecialDatesQuery>(CanViewSpecialDates);

        // POST /api/LeadTimeMatrix/SpecialDates
        builder.RegisterHandler<CreateSpecialDateCommand>(CanAddSpecialDates);

        // PUT /api/LeadTimeMatrix/SpecialDates/{id}
        builder.RegisterHandler<UpdateSpecialDateCommand>(CanEditSpecialDates);

        // DELETE /api/LeadTimeMatrix/SpecialDates/{id}
        builder.RegisterHandler<DeleteSpecialDateCommand>(CanDeleteSpecialDates);

        // GET /api/LeadTimeMatrix/SpecialDates/Categories
        builder.RegisterHandler<GetSpecialDateCategoriesQuery>(CanViewSpecialDates);

        // POST /api/LeadTimeMatrix/SpecialDates/Categories
        builder.RegisterHandler<CreateSpecialDateCategoryCommand>(CanAddSpecialDateCategories);

        // PUT /api/LeadTimeMatrix/SpecialDates/Categories/{id}
        builder.RegisterHandler<UpdateSpecialDateCategoryCommand>(CanEditSpecialDateCategories);

        // DELETE /api/LeadTimeMatrix/SpecialDates/Categories/{id}
        builder.RegisterHandler<DeleteSpecialDateCategoryCommand>(CanDeleteSpecialDateCategories);
    }

    /// <summary>
    /// Registers all Branding Departments authorization rules.
    /// </summary>
    private static void RegisterBrandingDepartmentsEndpoints(EndpointAuthorizationPolicyBuilder builder)
    {
        // GET /api/LeadTimeMatrix/BrandingDepartments
        builder.RegisterHandler<GetBrandingDepartmentsQuery>(CanViewBrandingDepartments);

        // Invoice Codes
        builder.RegisterHandler<CreateInvoiceCodeCommand>(CanAddInvoiceCodes);
        builder.RegisterHandler<UpdateInvoiceCodeCommand>(CanEditInvoiceCodes);
        builder.RegisterHandler<DeleteInvoiceCodeCommand>(CanDeleteInvoiceCodes);

        // Setup Codes
        builder.RegisterHandler<CreateSetupCodeCommand>(CanAddSetupCodes);
        builder.RegisterHandler<UpdateSetupCodeCommand>(CanEditSetupCodes);
        builder.RegisterHandler<DeleteSetupCodeCommand>(CanDeleteSetupCodes);

        // Additional Charges
        builder.RegisterHandler<CreateAdditionalChargeCommand>(CanAddAdditionalCharges);
        builder.RegisterHandler<UpdateAdditionalChargeCommand>(CanEditAdditionalCharges);
        builder.RegisterHandler<DeleteAdditionalChargeCommand>(CanDeleteAdditionalCharges);
    }

    /// <summary>
    /// Registers all Production Lead Time Matrix authorization rules.
    /// </summary>
    private static void RegisterProductionLeadTimesEndpoints(EndpointAuthorizationPolicyBuilder builder)
    {
        // GET /api/LeadTimeMatrix/Production
        builder.RegisterHandler<GetProductionLeadTimesQuery>(CanViewProductionLeadTimes);

        // Adjustments and Overrides
        builder.RegisterHandler<UpdateProductionAdjustmentCommand>(CanManageProductionAdjustmentsAndOverrides);
        builder.RegisterHandler<UpdateProductionOverrideCommand>(CanManageProductionAdjustmentsAndOverrides);

        // Personalisation Rules
        builder.RegisterHandler<CreateProductionPersonalisationRuleCommand>(CanAddProductionPersonalisationRules);
        builder.RegisterHandler<UpdateProductionPersonalisationRuleCommand>(CanEditProductionPersonalisationRules);
        builder.RegisterHandler<DeleteProductionPersonalisationRuleCommand>(CanDeleteProductionPersonalisationRules);

        // Groups
        builder.RegisterHandler<CreateProductionGroupCommand>(CanAddProductionGroups);
        builder.RegisterHandler<UpdateProductionGroupCommand>(CanEditProductionGroups);
        builder.RegisterHandler<DeleteProductionGroupCommand>(CanDeleteProductionGroups);

        // Quantity Breaks
        builder.RegisterHandler<CreateProductionQuantityBreakCommand>(CanAddProductionQuantityBreaks);
        builder.RegisterHandler<UpdateProductionQuantityBreakCommand>(CanEditProductionQuantityBreaks);
        builder.RegisterHandler<DeleteProductionQuantityBreakCommand>(CanDeleteProductionQuantityBreaks);
    }

    /// <summary>
    /// Registers all Warehouse Lead Time Matrix authorization rules.
    /// </summary>
    private static void RegisterWarehouseLeadTimesEndpoints(EndpointAuthorizationPolicyBuilder builder)
    {
        // GET /api/LeadTimeMatrix/Warehouse
        builder.RegisterHandler<GetWarehouseLeadTimesQuery>(CanViewWarehouseLeadTimes);

        // Adjustments and Overrides
        builder.RegisterHandler<UpdateWarehouseAdjustmentCommand>(CanManageWarehouseAdjustmentsAndOverrides);
        builder.RegisterHandler<UpdateWarehouseOverrideCommand>(CanManageWarehouseAdjustmentsAndOverrides);

        // Personalisation Rules
        builder.RegisterHandler<CreateWarehousePersonalisationRuleCommand>(CanAddWarehousePersonalisationRules);
        builder.RegisterHandler<UpdateWarehousePersonalisationRuleCommand>(CanEditWarehousePersonalisationRules);
        builder.RegisterHandler<DeleteWarehousePersonalisationRuleCommand>(CanDeleteWarehousePersonalisationRules);

        // Groups
        builder.RegisterHandler<CreateWarehouseGroupCommand>(CanAddWarehouseGroups);
        builder.RegisterHandler<UpdateWarehouseGroupCommand>(CanEditWarehouseGroups);
        builder.RegisterHandler<DeleteWarehouseGroupCommand>(CanDeleteWarehouseGroups);

        // Quantity Breaks
        builder.RegisterHandler<CreateWarehouseQuantityBreakCommand>(CanAddWarehouseQuantityBreaks);
        builder.RegisterHandler<UpdateWarehouseQuantityBreakCommand>(CanEditWarehouseQuantityBreaks);
        builder.RegisterHandler<DeleteWarehouseQuantityBreakCommand>(CanDeleteWarehouseQuantityBreaks);
    }
}
```

## Implementation Requirements

### Files to Create

1. **Claims Class**: `Moyo.Common.Domain/Claims/LeadTimeMatrix/LeadTimeMatrixClaims.cs`
2. **Authorization Registration**: `LeadTimeMatrix.Api/Authorization/LeadTimeMatrixAuthorizationRegistration.cs` (partial class with registration methods per feature area)

### Validation Rules

- Delete claims must be treated as sensitive and require explicit allocation
- Claims checks are enforced via `EndpointAuthorizationPolicyBuilder.RegisterHandler<THandler>(claim)`
- Use `RegisterHandlerWithoutClaims<THandler>()` only for endpoints that do not require authorization

---

## Acceptance Criteria

1. `LeadTimeMatrixClaims` class contains all 40 claims as const strings using `nameof()` pattern
2. `LeadTimeMatrixAuthorizationRegistration` registers all endpoint handlers with their corresponding claims
3. Each protected endpoint validates the appropriate claim before executing
4. Users without the required claim receive HTTP 403 Forbidden
5. Delete operations are restricted to users with explicit delete claims
6. Implementation follows the same pattern as `StockCheckingClaims` and `StockCheckingAuthorizationRegistration`
